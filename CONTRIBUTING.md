# Contributing

Thanks for contributing! A few guidelines to keep the project consistent.

- Create a feature branch: `feature/your-feature` or `fix/issue-description`.
- Follow existing coding style (use Prettier/ESLint in the frontend where configured).
- Keep changes focused and add tests where relevant.
- Open a PR describing the change and include screenshots for UI changes.

Security
- Do not commit secrets or `.env` files. Use environment variables for keys and tokens.
